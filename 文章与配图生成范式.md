# 文章与配图生成范式

适用于公众号系列文章，尤其是「数字货币量化」「交易认知」「策略风控」这类知识型内容。

## 一、生成顺序

必须按这个顺序执行：

1. 先写文章正文。
2. 从文章中提炼内容结构。
3. 再根据结构生成 PNG 信息图。
4. 最后把 PNG 插入文章开头。

不要先随便生成封面图。图片必须服务文章内容。

硬性要求：每篇文章写完后必须生成一张对应的 PNG 内容结构图，并插入文章开头；不能只生成正文。

双语硬性要求：每个主题必须同时生成中文文章和英文文章，并分别生成中文 PNG 结构图和英文 PNG 结构图。中文文章只能链接中文图片，英文文章只能链接英文图片，不能混用。

## 二、文章生成范式

### 1. 文章定位

文章不是广告文，也不是暴富文，而是：

- 认知教育
- 风险提醒
- 方法论建立
- 系列内容铺垫

语气要求：

- 适合公众号读者
- 对新手友好
- 有冲击力，但不夸张
- 不承诺收益
- 不构成投资建议

### 2. 推荐文章结构

```text
# 标题

开篇：用一个常见误区或痛点切入

提出核心问题：
这篇文章到底要回答什么？

正文：
一、原因/问题 1
二、原因/问题 2
三、原因/问题 3
……

底层本质：
把表面现象归纳成更深层逻辑

普通人该怎么做：
给出可执行建议

结尾：
升华主题，并承接后续系列

风险提示：
本文仅用于认知教育，不构成投资建议
```

### 3. 内容写作原则

- 先讲人性，再讲方法。
- 先讲风险，再讲收益。
- 先讲错误路径，再讲正确路径。
- 多用短句，适合手机阅读。
- 每一节只讲一个核心观点。
- 用列表增强可读性。
- 结尾要有一句可传播的金句。

示例金句：

```text
在市场里，先活下来的人，才有机会赚到后面的钱。
```

## 三、图片生成范式

### 1. 图片类型

图片不是普通封面，不是随机氛围图，而是：

```text
文章内容信息图 / 文章结构海报 / 公众号正文配图
```

要求：

- 必须是 PNG。
- 横图优先，推荐 1920 x 1080。
- 背景使用白色或浅色。
- 内容必须来自文章。
- 要能让读者一眼看懂文章框架。

### 2. 图片内容结构

一张图建议包含：

- 大标题
- 核心副标题
- 主要原因列表
- 错误认知 vs 真实结果
- 正确路径 / 方法论
- 新手建议
- 结论金句
- 风险提示

以《为什么90%的人做数字货币最后亏钱》为例：

```text
主标题：
为什么90%的人做数字货币最后亏钱

副标题：
不是市场不给机会，而是你用错了方法

右侧列表：
亏钱的9大原因

中部模块：
赚钱时的错觉 VS 亏钱后的现实

底部模块：
正确的交易思维
认知 > 风控 > 系统 > 执行 > 复盘 > 长期生存

新手建议：
先学习，再实战
小资金，低风险开始
不用高杠杆
建立自己的交易系统

结论：
先活下来，才有机会赚到后面的钱
```

### 3. 图片视觉风格

推荐风格：

- 白色或浅米色背景
- 金色强调数字和重点
- 红色表示亏损、风险、错误
- 绿色表示正确路径、风控、建议
- 金融图表元素作为背景辅助
- 使用卡片分区，但不要太碎
- 信息密度高，但要清晰

避免：

- 黑色背景，除非明确要求
- 纯氛围图
- 只有 K 线和人物，没有文章结构
- 夸张暴富元素
- 真实交易所 Logo
- 具体币种 Logo
- 收益承诺
- 过多小字

## 四、推荐文件组织

```text
articles/
  day01-文章标题.zh.md
  day01-article-title.en.md

assets/
  day01-article-infographic.zh.png
  day01-article-infographic.en.png
```

说明：

- Markdown 保存正文，中文文章和英文文章分开保存。
- PNG 是最终发布用图，中文图和英文图分开保存。
- 中文文章开头插入中文 PNG。
- 英文文章开头插入英文 PNG。

中文文章引用图片：

```markdown
![文章标题](../assets/day01-article-infographic.zh.png)
```

英文文章引用图片：

```markdown
![Article title](../assets/day01-article-infographic.en.png)
```

## 五、标准执行流程

### Step 1：生成文章

输入：

```text
标题：为什么90%的人做数字货币最后亏钱
阶段：认知基础
目标读者：数字货币新手、想学习量化的人
```

输出：

```text
articles/day01-为什么90%的人做数字货币最后亏钱.zh.md
articles/day01-why-90-percent-of-crypto-traders-lose-money.en.md
```

### Step 2：提炼结构

从文章中提炼：

- 文章主问题
- 核心观点
- 主要原因
- 方法论路径
- 结论金句

### Step 3：生成 PNG 信息图

输出：

```text
assets/day01-article-infographic.zh.png
assets/day01-article-infographic.en.png
```

规格：

```text
尺寸：1920 x 1080
格式：PNG
背景：白色或浅色
用途：公众号文章首图 / 正文信息图
语言：中文图片只放中文，英文图片只放英文
```

### Step 4：插入文章

在中文文章标题下方插入中文图片：

```markdown
![为什么90%的人做数字货币最后亏钱](../assets/day01-article-infographic.zh.png)
```

在英文文章标题下方插入英文图片：

```markdown
![Why 90% of Crypto Traders Lose Money](../assets/day01-article-infographic.en.png)
```

## 六、下次可直接使用的提示词

```text
请按我们的公众号文章生成范式处理这个主题：

标题：{文章标题}
Day：{第几天}
阶段：{阶段名称}

要求：
1. 先生成一篇中文公众号文章和一篇英文文章，分别保存到 articles/。
2. 文章要适合新手阅读，强调认知、风险和方法论，不做收益承诺。
3. 写完文章后，分别从中文文章和英文文章中提炼内容结构。
4. 再分别生成中文白色背景 PNG 信息图和英文白色背景 PNG 信息图，保存到 assets/。
5. 图片尺寸优先 1920x1080，必须是 PNG。
6. 图片要包含文章主标题、核心观点、主要原因/步骤、正确路径、结论金句和风险提示。
7. 图片不是随机封面，要像文章内容架构海报。
8. 中文文章只能插入中文 PNG，英文文章只能插入英文 PNG。
```

## 七、质量检查清单

文章检查：

- 标题是否明确
- 开篇是否有痛点
- 正文是否分层清晰
- 是否有可执行建议
- 是否有风险提示
- 是否避免收益承诺

图片检查：

- 是否为 PNG
- 是否白色或浅色背景
- 是否引用文章内容
- 中文文章是否链接中文图片
- 英文文章是否链接英文图片
- 是否包含主要结构
- 字是否清晰可读
- 是否适合公众号展示
- 是否没有交易所 Logo 和收益承诺
